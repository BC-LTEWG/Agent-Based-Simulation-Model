# Simulation of Trade Between Capitalist and Labor-Time Economies

## The Event-Based Simulation: A Dynamic Capitalist Economy Backend for the Trade Simulation
### Description / Overview

The Event-Based Simulation implements a dynamic capitalist economy model designed to serve as the price- and supply-generating backend for this trade interface. Its purpose is to generate internally consistent trajectories of prices, outputs, supplies, labor usage, and value magnitudes under capitalist production relations, which can then be used by this trading system. It is built as a production-grounded and class-structured model. 

### Simulation Design

- Constructs a multi-commodity capitalist economy using a generated input–output matrix and labor coefficients
- Normalizes technical coefficients and the labor vector to ensure internally consistent production costs
- Initializes prices, supplies, outputs, wages, profits, and inventories from parameterized initial conditions
- Evolves the economy forward in discrete time by repeatedly applying a single event-based update step
- Records trajectories of prices, supplies, and related state variables at each time step
- Detects equilibrium by identifying a time step with no change in prices and supplies relative to the previous step
- Upon equilibrium detection, enables externally specified random events for a fixed post-equilibrium window (40 time steps)
- Applies the effects of enabled events directly to the economic state during simulation
- Exports full trajectory data with equilibrium annotations to CSV for downstream analysis
- Produces visualizations of individual commodity prices, supplies, and aggregate averages

### Theoretical Frameworks Used

- Leontief input-output model
- Classical political economy
- Marxian value theory

## The Trade Interface 
### Description / Overview

The Trade Interface aims to implement a discrete-time simulation of exchange between a Capitalist Economy (CE) and a Labor-Time Economy (LTE). Its role is to mediate trade between two economies operating under distinct value systems by identifying shortages and surpluses, constructing import and export demands, and executing trades based on value equivalence. The interface consumes price, supply, and equilibrium information generated by the capitalist economy simulation and applies the resulting trade effects back onto the capitalist economy, while treating the labor-time economy as a needs-driven trading partner. The trade process is recomputed independently at each time step and does not assume persistent trade relations across periods.

### Simulation Design

- Evaluates the commodity states of both economies at each time step to identify missing products and unmet needs
- Constructs import demand lists for each economy based on detected shortages
- Constructs export supply lists for the capitalist economy based on detected surpluses
- Removes import and export entries automatically when shortages or surpluses disappear
- Restricts the labor-time economy to trading only when it is missing required goods
- Treats overproduction in the labor-time economy as conditional on trade necessity
- Computes desired trade magnitudes in value terms rather than physical quantities
- Converts labor-time values into monetary equivalents using a dynamic MELT parameter
- Applies a trade-matching algorithm to determine feasible exchanges subject to value constraints
- Executes trades only when mutually compatible value bundles can be assembled
- Applies the effects of completed trades directly to the capitalist economy’s state
- Recomputes trade demands and opportunities independently at each subsequent time step
